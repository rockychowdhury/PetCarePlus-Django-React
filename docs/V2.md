## 1. Project Overview

**PetCarePlus** is a bilingual (Bangla/English), location-scoped pet and livestock care platform built for the Bangladeshi context. It is not a generic global pet marketplace. Every feature, scope decision, and UX choice is driven by the real needs of users in Bangladesh — companion pet owners in cities, farmers in rural areas, and the vets and care providers who serve both.

### What the platform does

- Provides **publicly accessible guidelines** for companion pets (cat, dog, rabbit, bird) and livestock (cow, goat, chicken, duck) in both Bangla and English
- Maintains a **vaccination and medicine reference** for all supported animals, searchable and filterable
- Lists **government programs, NGO resources, and emergency contacts** scoped to the user's district
- Connects users to **verified local service providers** (vets, groomers, sitters, trainers, pharmacies) within their local network circle
- Offers an **AI-powered care assistant** that asks questions, diagnoses issues, recommends home care, assigns urgency, and suggests local providers to call
- Provides a **free rehoming service** for cats and dogs only, within the local network
- Supports **service booking and reviews** for all companion pet services

### What it is NOT

- It is not a national marketplace trying to serve everyone everywhere
- It is not a social media platform for pet photos
- Rehoming is a secondary feature — service access is the core value

## 2. Why We're Rebuilding

The v1 project (`PetCarePlus-Django-React`) had the right stack but no unique value proposition. It was a generic CRUD platform that could have been built for any country with any pets. The problems:

| v1 Problem | v2 Solution |
|---|---|
| English-only interface | Bangla + English with language preference per user |
| No geographic scoping | Local network circle — district/upazila-first queries |
| Generic pet types, no restrictions | Two-tier animal system with enforced scope rules |
| No AI feature | AI diagnostic assistant with provider routing |
| Render memory limit causes server downtime | Move to Railway/Fly.io, proper worker config |
| No farming support | Livestock guidelines and vaccination info (cow, goat, chicken, duck) |
| Rehoming was the main feature | Services are the main feature; rehoming is free but secondary |
| No real-world differentiation | Solves a genuine gap in the Bangladeshi market |

## 3. Core Philosophy and Focus

### Local first
Every query for providers, rehoming listings, and AI suggestions defaults to the user's `district` first. If no matches exist, it expands to `division`. The platform works on a **local network circle** — users are matched with what is actually reachable and usable in their area.

### Services are the priority
The platform's core value is connecting people to **verified, reviewed local care providers** and giving them **the knowledge to make informed decisions** before, during, and after that interaction. Rehoming is available and free, but it is not the headline feature.

### AI before manual search
Instead of making users browse providers and guess who to call, the AI assistant **qualifies the problem first**. It asks about symptoms, duration, animal age, and behaviour, then tells the user:
- What the likely issue is
- What they can do at home right now
- Whether this is an emergency
- Which type of provider they need
- Specific local providers matching that need (ranked by rating + proximity)

### Bangla is the primary language
UI defaults to Bangla. All content has Bangla variants. The AI assistant converses in Bangla or English based on the user's preference. Error messages, form labels, notifications — everything is translatable.

### Scope discipline
Not all pets. Not all services. Not all of Bangladesh. Defined lists, enforced at the database level, prevent scope creep and keep the platform usable and maintainable.

## 5. Animal Tier System

Animals are divided into two tiers. This is enforced at the **database level** via `animal_types.category` and boolean flags, not just application logic.

### Tier 1 — Companion pets (full platform support)

| Animal | Bangla | Rehoming | Services | Guidelines | Vaccination |
|---|---|---|---|---|---|
| Cat | বিড়াল | ✅ (free) | ✅ | ✅ | ✅ |
| Dog | কুকুর | ✅ (free) | ✅ | ✅ | ✅ |
| Rabbit | খরগোশ | ❌ | ✅ | ✅ | ✅ |
| Bird | পাখি | ❌ | ✅ | ✅ | ✅ |

### Tier 2 — Livestock (information only)

| Animal | Bangla | Rehoming | Services | Guidelines | Vaccination |
|---|---|---|---|---|---|
| Cow | গরু | ❌ | ❌ | ✅ | ✅ |
| Goat | ছাগল | ❌ | ❌ | ✅ | ✅ |
| Chicken | মুরগি | ❌ | ❌ | ✅ | ✅ |
| Duck | হাঁস | ❌ | ❌ | ✅ | ✅ |

### Enforcement rules

- `pets` table: CHECK constraint allows only `category = 'companion'` animal types
- `rehoming_listings` table: CHECK constraint allows only `supports_rehoming = TRUE` animal types
- `provider_animal_types`: providers can only link to animals where `supports_services = TRUE`
- API serializers validate animal type before accepting any pet/listing/service creation

## 6. Local Network Architecture

### How location works

Bangladesh has a clear administrative hierarchy:

```
Division (বিভাগ)
  └── District (জেলা)
        └── Upazila (উপজেলা)
```

Users store `division`, `district`, and `upazila` on registration. No latitude/longitude is stored or required. This is intentional — it is simpler, faster, and appropriate for the actual geography of the user base.

### Query scoping rules

When fetching providers, rehoming listings, or AI suggestions, the backend applies a **cascade scope**:

1. First: match `district` (same district as user)
2. If results < threshold (configurable, default 3): expand to `division`
3. If still insufficient: return all verified results with a "showing results beyond your area" label

### Implementation

```python
# Example: local-scoped provider queryset
def get_local_providers(user, provider_type=None, animal_type_id=None):
    qs = ServiceProvider.objects.filter(is_verified=True, is_active=True)

    if provider_type:
        qs = qs.filter(provider_type=provider_type)
    if animal_type_id:
        qs = qs.filter(provider_animal_types__animal_type_id=animal_type_id)

    local = qs.filter(district=user.district)
    if local.count() >= 3:
        return local.order_by('-avg_rating')

    regional = qs.filter(division=user.division)
    if regional.count() >= 3:
        return regional.order_by('-avg_rating')

    return qs.order_by('-avg_rating')
```

### Anonymous users

Users who have not logged in do not have a stored location. In this case:
- Guidelines and vaccination info are shown without location filter
- Govt resources prompt the user to select a district from a dropdown
- Provider directory shows a "Select your area" prompt
- AI assistant asks for location as the first question if it needs to suggest providers

## 7. Database Schema Reference

The full SQL schema is in `docs/schema.sql`. This section summarises the table purposes and key relationships.

### Tables overview

| Table | Purpose |
|---|---|
| `users` | All users — pet owners, farmers, providers, admins. Stores location (division/district/upazila) and language preference |
| `animal_types` | Seed reference table. Defines the 8 supported animals, their tier (companion/livestock), and capability flags |
| `guidelines` | Bilingual care guides. Filterable by animal, topic, season. Full-text search via `tsvector` trigger |
| `vaccination_records` | Vaccine schedules, dosage, disease prevention, local medicine names per animal |
| `govt_resources` | Government offices, NGOs, emergency contacts — scoped to division/district |
| `service_providers` | Provider business profiles with location, type, verification status, aggregated rating |
| `provider_services` | Individual services offered by a provider with pricing |
| `provider_animal_types` | Many-to-many: which animals each provider treats |
| `bookings` | Appointment records linking user, provider, service, and pet |
| `reviews` | One review per completed booking. Triggers auto-update provider avg_rating |
| `ai_sessions` | Full conversation history (JSONB) + extracted urgency, diagnosis, care advice |
| `ai_provider_suggestions` | Providers suggested by AI at end of session with ranking and reason |
| `pets` | Companion pet profiles (only `category = 'companion'` allowed) |
| `rehoming_listings` | One listing per pet. Cat and dog only. Free within local network |
| `rehoming_applications` | Applications to adopt a listed pet |
| `notifications` | In-app notifications for bookings, applications, reviews |

### Key design decisions

**Dual-language columns:** Every user-facing text field exists in `_en` and `_bn` variants. The API serializer returns the appropriate variant based on `request.user.preferred_language` (or `Accept-Language` header for anonymous users).

**JSONB for AI conversations:** `ai_sessions.conversation_history` stores the full multi-turn conversation as a JSONB array. Extracted fields (`urgency_level`, `ai_diagnosis_summary`, `ai_care_advice`) are written when the session concludes so analytics queries never need to parse JSONB.

**Aggregated rating via trigger:** `service_providers.avg_rating` and `total_reviews` are kept in sync by a PostgreSQL trigger on `reviews`. Provider list queries do not need a subquery.

**Soft deletes:** Users (`is_active`), pets (`is_active`), providers (`is_active`), listings (`status = 'withdrawn'`). Hard deletes only on junction tables.

### Key patterns

#### Language-aware serializer mixin

```python
# common/mixins.py
class BilingualMixin:
    """
    Returns _en or _bn field variants based on user preference.
    Apply to any serializer that has bilingual fields.
    """
    def get_bilingual_field(self, obj, field_name):
        request = self.context.get('request')
        lang = 'bn'
        if request and request.user.is_authenticated:
            lang = request.user.preferred_language
        elif request:
            lang = request.headers.get('Accept-Language', 'bn')[:2]
        return getattr(obj, f'{field_name}_{lang}', None) or getattr(obj, f'{field_name}_en', None)
```

#### Custom permissions

```python
# common/permissions.py
class IsProvider(permissions.BasePermission):
    def has_permission(self, request, view):
        return request.user.is_authenticated and request.user.role == 'provider'

class IsVerifiedProvider(permissions.BasePermission):
    def has_permission(self, request, view):
        return (
            request.user.is_authenticated
            and request.user.role == 'provider'
            and hasattr(request.user, 'service_provider')
            and request.user.service_provider.is_verified
        )

class IsOwnerOrAdmin(permissions.BasePermission):
    def has_object_permission(self, request, view, obj):
        if request.user.role == 'admin':
            return True
        return obj.owner == request.user or obj.user == request.user
```

## 9. AI Assistant Feature

The AI assistant is the platform's most important feature. It is the primary entry point for users with a problem who do not know where to start.

### Conversation flow

```
User opens AI assistant
        │
        ▼
Step 1: Animal selection
  → "Which animal are you asking about?"
  → Shown as visual buttons (cat, dog, cow, goat...)
        │
        ▼
Step 2: Problem description
  → "Describe what you're noticing. You can write in Bangla or English."
        │
        ▼
Step 3: AI clarification (2–3 questions)
  → "How long has this been going on?"
  → "Is the animal eating and drinking?"
  → "How old is the animal approximately?"
        │
        ▼
Step 4: AI diagnosis output
  → Likely cause (explained in plain Bangla or English)
  → Home care steps (what to do right now)
  → Urgency level: monitor / see vet this week / call now / emergency
        │
        ▼
Step 5: Provider suggestion (if vet recommended)
  → Ranked list of local providers
  → Reason for each suggestion
  → Contact button
```

### Urgency levels

| Level | Label (English) | Label (Bangla) | Colour |
|---|---|---|---|
| `monitor_at_home` | Monitor at home | বাড়িতে পর্যবেক্ষণ করুন | Green |
| `see_vet_this_week` | See a vet this week | এই সপ্তাহে পশু চিকিৎসক দেখান | Yellow |
| `call_vet_now` | Call a vet now | এখনই পশু চিকিৎসক ডাকুন | Orange |
| `emergency` | Emergency — act immediately | জরুরি অবস্থা | Red |

### Backend implementation

```python
# apps/ai_assistant/views.py

class AIChatView(APIView):
    permission_classes = [IsAuthenticatedOrReadOnly]

    def post(self, request):
        session_id = request.data.get('session_id')
        user_message = request.data.get('message')
        language = request.data.get('language', 'bn')

        session = get_or_create_session(session_id, request.user)
        history = session.conversation_history

        system_prompt = self.build_system_prompt(session, language)
        history.append({'role': 'user', 'content': user_message})

        response = call_gemini(system_prompt, history)

        history.append({'role': 'assistant', 'content': response['text']})
        session.conversation_history = history
        session.total_turns += 1

        if response.get('session_complete'):
            session.urgency_level = response['urgency_level']
            session.ai_diagnosis_summary = response['summary']
            session.ai_care_advice = response['care_advice']
            session.ended_at = now()
            self.generate_provider_suggestions(session)

        session.save()
        return Response(response)

    def build_system_prompt(self, session, language):
        lang_instruction = "Respond in Bangla (বাংলা)." if language == 'bn' else "Respond in English."
        return f"""
You are a compassionate, knowledgeable animal care assistant for Bangladesh.
{lang_instruction}

Your role:
1. Ask 2-3 targeted questions to understand the problem
2. Provide a clear, plain-language explanation of the likely issue
3. Give practical home care steps the owner can take right now
4. Assign an urgency level: monitor_at_home | see_vet_this_week | call_vet_now | emergency
5. If urgency is see_vet_this_week or higher, indicate that you will suggest local providers

Always be warm and reassuring. Avoid medical jargon. 
When the diagnosis is complete, output a JSON block at the end:
{{
  "session_complete": true,
  "urgency_level": "...",
  "summary": "...",
  "care_advice": "...",
  "needs_provider": true/false
}}
"""
```

### Provider suggestion algorithm

When `session_complete = true` and `needs_provider = true`:

1. Get the animal type from the session
2. Map `urgency_level` to required `provider_type` (e.g., `call_vet_now` → `vet`)
3. Run local network query for matching providers
4. Rank by: `(avg_rating × 0.6) + (is_verified × 0.3) + (total_reviews_normalised × 0.1)`
5. Store top 3 in `ai_provider_suggestions`
6. Return with `reason_en` and `reason_bn` for each

## 12. UI Theme and Design System

### Design direction

The UI is clean, warm, and approachable. It is not a clinical healthcare dashboard and not a flashy marketplace. The primary user base includes rural farmers and urban pet owners — the design must feel trustworthy and easy to scan in both Bangla and English.

### Colour palette

| Token | Hex | Usage |
|---|---|---|
| `--color-primary` | `#2D6A4F` | Primary buttons, active states, links |
| `--color-primary-light` | `#52B788` | Hover states, highlights |
| `--color-primary-muted` | `#D8F3DC` | Background tints, selected cards |

- Refresh token: 7 days
- Stored in memory (access) and httpOnly cookie (refresh) — no localStorage for tokens

### User roles

| Role | Description |
|---|---|

| `provider` | Can register a provider profile, manage bookings, respond to reviews |

A user can be both `pet_owner` and `provider` — the role field stores the primary role. The provider relationship is checked separately via `request.user.service_provider`.

## 15. Internationalisation (i18n)

### Approach

Two-track system:

1. **UI strings** — static labels, buttons, error messages stored in `i18n/bn.json` and `i18n/en.json`. Switched globally based on `authStore.language`.

2. **Content fields** — dynamic content from the API has `_en` and `_bn` variants. The `useLanguage()` hook returns the current language, and a helper picks the right field:

```javascript
// hooks/useLanguage.js
export function useField(obj, field) {
  const { lang } = useLanguage();
  return obj?.[`${field}_${lang}`] || obj?.[`${field}_en`] || '';
}

// Usage in a component
const title = useField(guideline, 'title'); // returns title_bn or title_en
```

### Bangla font loading

## 16. Deployment and Infrastructure

### Problem with v1 (Render)

The v1 deployment on Render free tier suffered from memory limit errors causing automatic restarts. The root cause was running Django, Celery, and Redis in a configuration that exceeded the 512MB memory limit of the free instance.

### v2 deployment plan

| Service | Platform | Notes |
|---|---|---|
| Django API | Railway (Starter) | 512MB RAM, 1 vCPU — sufficient without Celery on same dyno |
| PostgreSQL | Railway managed DB | Automatic backups, connection pooling |
| React frontend | Vercel | Free tier, CDN-distributed, automatic deploys from `main` |
| Static files | Cloudflare R2 | Media uploads (pet photos, provider avatars) |

### Removing Celery dependency from critical path

Celery is used only for:
- Sending notification emails (non-critical, can be async via Django background tasks)
- Scheduled cleanup of expired AI sessions

For v2, use `django-background-tasks` instead of Celery+Redis to eliminate the worker process memory overhead. Celery can be reintroduced later when traffic justifies it.

## 19. Recruiter Talking Points

When presenting this project, these are the decisions worth explaining:

### Product decisions (show you think beyond code)

**"Why did you restrict it to 8 animals?"**  
Scope discipline. A platform that claims to cover all animals covers none well. Defining a list and enforcing it at the database level — not just in the UI — shows that the constraints are real, not cosmetic.

**"Why local network scope instead of a national marketplace?"**  
In rural Bangladesh, a vet 50km away is effectively inaccessible. Defaulting every query to the user's district makes the platform actually useful. The cascade fallback to division when local results are sparse shows that the system degrades gracefully rather than showing nothing.

**"Why is rehoming a secondary feature if it was the original centrepiece?"**  
Because scope interviews show that the real pain point is access to care, not rehoming. Rehoming is free and available, but services are where users return repeatedly. This is a product insight, not a code insight.

### Technical decisions (show depth)

**"Why Bangla as the primary language?"**  
The actual users — farmers, rural pet owners — use Bangla. Building English-first and bolting on Bangla later produces bad translations and worse UX. The dual-column database design (`title_en` / `title_bn`) and the language-aware serializer mixin are both built for this from the start.

**"Why did you drop Celery from the critical path?"**  
The v1 server was crashing due to Render's 512MB memory limit. Running Django, Celery, and Redis together on a free instance is the cause. Replacing Celery with `django-background-tasks` for non-critical async work eliminates the worker process overhead and makes the app stable for demo. This is an infrastructure decision driven by real constraints.

**"What's interesting about the AI assistant technically?"**  
The conversation history is stored as JSONB in PostgreSQL, but the urgency level, diagnosis summary, and care advice are extracted as typed columns when the session completes. This means analytics and reporting queries never need to parse JSONB, while the full conversation remains available for support and debugging. The provider suggestion at the end of a session runs a weighted ranking algorithm combining rating, verification status, and review count.

**"Why not use Haversine / PostGIS for proximity?"**  
Bangladesh's administrative hierarchy (division → district → upazila) is precise enough for this use case and far simpler to implement, query, and explain. PostGIS adds complexity without adding value when users are already identified by their district. The cascade query logic (district → division → all) handles the edge cases cleanly.
